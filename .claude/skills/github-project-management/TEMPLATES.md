# Issue テンプレート

## 親Issue（Epic）テンプレート

```markdown
## 概要
[機能の全体的な説明]

## 画面レイアウト / 設計図
[必要に応じてASCIIアートや図を含める]

## サブタスク（実装順・依存関係付き）

\`\`\`
#A  基盤設定
 ↓
#B  前提変更
 ↓
#C  コア実装
 ↓
#D  統合（最後）
\`\`\`

- [ ] #A [タスク1の説明]
- [ ] #B [タスク2の説明]（#Aの後）
- [ ] #C [タスク3の説明]（#Bの後）
- [ ] #D [タスク4の説明]（#Cの後・最後）

## 技術仕様

### 条件付きコンパイル
- `cfg(feature = "[feature-name]")` で制御

### ビルド方法
\`\`\`bash
[ビルドコマンド]
\`\`\`

## 主要な構造体/関数

\`\`\`rust
// コード例
\`\`\`

## 変更ファイル一覧

| ファイル | 変更内容 |
|---------|---------|
| `path/to/file1` | 新規作成 |
| `path/to/file2` | 修正 |

## 参照ファイル（既存の実装パターン）

| ファイル | 参照ポイント |
|----------|--------------|
| `path/to/reference` | 既存パターンの説明 |
```

## サブIssueテンプレート

```markdown
## 親Issue
#[親Issue番号]

## 依存関係
- なし（最初に実施）
または
- #[前のIssue番号] 完了後に実施

## タスク
[具体的なタスク内容]

## 変更ファイル
- `path/to/file`

## 実装内容

\`\`\`rust
// 具体的なコード例
\`\`\`

## 完了条件
- [ ] [具体的な実装条件1]
- [ ] [具体的な実装条件2]
- [ ] `cargo check` がエラーなく通る
- [ ] （該当する場合）テストが通る
```

## 使用例

### 新機能追加の場合

親Issue:
```
feat: ユーザー認証機能
```

サブIssue:
```
feat(auth): データベーススキーマ追加
feat(auth): 認証API実装
feat(auth): フロントエンド連携
feat(auth): テスト追加
```

### バグ修正の場合

親Issue:
```
fix: ログイン時のセッション管理不具合
```

サブIssue:
```
fix(session): セッションタイムアウト修正
fix(session): トークン更新ロジック修正
```
